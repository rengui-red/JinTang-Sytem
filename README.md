\# 🛡️ 金汤 (JinTang) - 智能端点防御与流束识别系统



> \*\*金汤\*\*：取“固若金汤，稳如泰山。”之意。

旨在构建一个，能够识别，恶意信息集束，束流量特征、阻断远程控制的信息束流通道。
迅速反应，上传下达，形成本地与云端，迅速联动，阻断恶意束流侵害的本地防御原型。



\## 🎯 项目愿景


本项目，致力于解决传统防御难以识别\*\*“隐蔽远程控制（C2）流束”\*\*的痛点。

不仅仅是拦截病毒，更是要识别出\*\*“非本人、非本机”\*\*的异常操作意图。
通过本地熔断与云端联动，构建主动防御体系。对恶意侵害的信息“集束流”，迅速上传，主动激发云端防御识别，升级生物身份验证，识别与核验，非本人，非本机特征，实施脑机验证。识别脑~电脑，机~手机，非本人，非本机，异常信息束流，即刻启动“上传下达”阻断模式。



\## 🧠 核心防御逻辑



\### 1. 识别“信息流束” (Stream Recognition)

黑客控制，往往伪装成正常流量。金汤不只看内容，更看\*\*行为特征\*\*：

\- \*\*心跳检测\*\*：识别程序是否在固定间隔（如60秒）发送固定大小的数据包（典型远控特征）。

\- \*\*流量指纹\*\*：分析上下行流量的不对称性，识别伪装成 DNS 或 HTTPS 的隧道流量。

\- \*\*行为基线\*\*：建立用户操作习惯（鼠标轨迹、按键节奏），区分“真人”与“脚本”。



\### 2. 迅速反应与阻断 (Rapid Response)

一旦识别出恶意流束，系统立即执行“熔断机制”：

1\. \*\*本地阻断\*\*：调用系统底层 API 切断可疑进程的网络连接。

2\. \*\*进程挂起\*\*：暂停可疑程序的运行，防止进一步破坏。



\### 3. 云端联动 (Cloud Linkage)

\- \*\*威胁上报\*\*：将恶意流束的指纹（IP、特征码）上传至云端大脑。

\- \*\*全网免疫\*\*：云端分析后下发黑名单，让所有金汤节点瞬间获得对该威胁的免疫力。



\## 🛠️ 技术栈

\- \*\*核心语言\*\*: Python (模拟与原型开发)

\- \*\*流量分析\*\*: Scapy / Socket

\- \*\*系统交互\*\*: OS / Subprocess

\- \*\*部署目标\*\*: Linux (Ubuntu/CentOS) / Cloud Server

### 🚀 如何运行

#### 方法一：使用 Docker 运行（推荐）

这是最简单、最不容易出错的方式。

1.  **构建镜像**
    在项目根目录下执行：
    ```bash
    docker build -t jintang-waf .
    ```

2.  **启动容器**
    ```bash
    docker run -d -p 80:80 --name jintang jintang-waf
    ```

#### 方法二：使用本地脚本运行

如果你已经安装了 OpenResty 或 Nginx，可以使用自带的 `run.sh` 脚本。

1.  **赋予执行权限**
    ```bash
    chmod +x run.sh
    ```

2.  **执行启动**
    ```bash
    ./run.sh
    ```)\*
